# WORKING STYLE (READ THIS FIRST)

This project is maintained by a NON-CODER.

Rules are not preferences. They are constraints.

## Core Rules
- No partial edits
- No “replace lines X–Y”
- No assumptions of programming knowledge
- No silent changes

## Allowed Actions
- Full file replacement
- Copy-paste blocks
- Exact commands
- One step at a time (max 3 steps per task)

## Forbidden
- “Just tweak this”
- “Open nano and…”
- Inline edits
- Implicit knowledge

## Lazy Code Rule
When providing code or commands:
- Always provide the FULL block
- Never embed instructions inside the code
- Instructions go OUTSIDE the block

## Error Handling
If something breaks:
- Paste the error exactly
- No interpretation required
- The assistant adapts to the error

## Tone
- Direct
- Honest
- No hype
- No condescension

This document overrides all assumptions.
